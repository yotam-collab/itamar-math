# WORD HACK — Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build WORD HACK — a 3-phase association game (Discovery → Rate & Create → Quiz) with a Cloudflare Worker + D1 backend for storing/rating/sharing associations.

**Spec:** `docs/superpowers/specs/2026-04-15-word-hack-association-game-design.md`

**Architecture:** New Cloudflare Worker (`cloudflare-associations-worker`) with D1 SQLite database. React game component follows existing patterns: `GameShell` portal wrapper, `useGameSession` for XP/combo, `useWordSelector` for word pool, `useSessionQueue` for re-queuing. 3-phase state machine: all 10 words pass through Phase 1, then all through Phase 2, then all through Phase 3.

**Tech Stack:** React 19, TypeScript, Zustand, TailwindCSS. Cloudflare Workers + D1 for backend. Existing design tokens from `constants.ts` (BG=#0F0B1E, ACCENT=#818CF8, etc.)

**Existing patterns to follow:**
- Worker structure: `cloudflare-tts-worker/` (worker.js + wrangler.toml)
- Game components: GameShell portal, useGameSession, useWordSelector
- Game registration: `types.ts` GameId → `constants.ts` GAME_CONFIGS → `GameRouter.tsx` lazy import + switch case
- Coach missions: `mockCoachData.ts` mission creation with unlockOrder + route/routeParams
- Word images: `src/utils/wordImages.ts` → `getWordImageUrl(wordId)`, `hasWordImage(wordId)`

- [ ] **Step 2: Create schema.sql**

Exactly as specified in the design doc:

```sql
CREATE TABLE associations (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  word_id INTEGER NOT NULL,
  english TEXT NOT NULL,
  text TEXT NOT NULL,
  source TEXT NOT NULL,   -- 'original' | 'ai' | 'student'
  author_id TEXT,
  author_name TEXT,
  status TEXT NOT NULL DEFAULT 'active',
  avg_rating REAL DEFAULT 0,
  rating_count INTEGER DEFAULT 0,
  created_at TEXT DEFAULT (datetime('now')),
  UNIQUE(word_id, text)
);

CREATE TABLE ratings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  association_id INTEGER NOT NULL REFERENCES associations(id),
  user_id TEXT NOT NULL,
  rating INTEGER NOT NULL CHECK(rating BETWEEN 1 AND 5),
  created_at TEXT DEFAULT (datetime('now')),
  UNIQUE(association_id, user_id)
);

CREATE TABLE moderation_log (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  association_id INTEGER REFERENCES associations(id),
  action TEXT NOT NULL,
  reason TEXT,
  matched_pattern TEXT,
  created_at TEXT DEFAULT (datetime('now'))
);

CREATE INDEX idx_assoc_word ON associations(word_id, status);
CREATE INDEX idx_assoc_status ON associations(status, avg_rating DESC);
CREATE INDEX idx_ratings_assoc ON ratings(association_id);
```

- [ ] **Step 3: Implement worker.js with all endpoints**

The worker handles CORS, routing, and all 6 endpoints:

| Method | Path | Auth | Description |
|--------|------|------|-------------|
| GET | `/associations/:wordId` | None | Single word's associations (sorted by rating, originals first) |
| GET | `/associations/batch?ids=1,2,3` | None | Batch fetch for 10 words (Phase 1 preload) |
| POST | `/associations` | None | Create student association (with content moderation) |
| POST | `/ratings` | None | Rate an association (upsert, recalculates avg) |
| GET | `/promoted` | None | All promoted associations (rating_count >= 10, avg >= 3.5) |
| POST | `/admin/import` | Bearer token | Bulk import original/AI associations |

**Content moderation in POST /associations:**
- Server-side banned word list (~200 Hebrew + English terms)
- Pattern matching for l33t speak substitutions
- If flagged: `status = 'pending_review'`, log to `moderation_log`
- If clearly offensive: `status = 'rejected'`, log reason

**Rating upsert in POST /ratings:**
- INSERT OR REPLACE on (association_id, user_id) unique constraint
- After insert: recalculate `avg_rating` and `rating_count` on the association row atomically
- Check if association now qualifies for promotion (count >= 10, avg >= 3.5) and update status to 'promoted'

**Verification:** Deploy with `wrangler deploy`, run schema with `wrangler d1 execute znk-associations --file=schema.sql`. Test each endpoint with curl.

- [ ] **Step 1: Create association API service**

```typescript
// src/services/associationApi.ts

const API_BASE = 'https://znk-associations.<account>.workers.dev'
// Use import.meta.env.VITE_ASSOCIATIONS_API or fallback

interface Association {
  id: number
  word_id: number
  english: string
  text: string
  source: 'original' | 'ai' | 'student'
  author_id: string | null
  author_name: string | null
  status: string
  avg_rating: number
  rating_count: number
}

interface WordAssociations {
  associations: Association[]
  originalId: number | null
}

// Batch fetch with 24h localStorage cache
export async function fetchAssociationsBatch(wordIds: number[]): Promise<Record<number, WordAssociations>>

// Rate an association (fire-and-forget)
export function rateAssociation(associationId: number, userId: string, rating: number): void

// Submit student-created association
export async function submitAssociation(wordId: number, text: string, authorId: string, authorName: string): Promise<{ association: Association; moderation: 'approved' | 'flagged' }>

// Get promoted associations (12h cache)
export async function fetchPromoted(): Promise<Record<number, string>>
```

**Key details:**
- `getUserId()` helper: reads/creates UUID from `localStorage.getItem('znk-user-id')`
- Cache keys: `znk-assoc-batch-{hash}` (24h TTL), `znk-assoc-promoted` (12h TTL)
- `rateAssociation` is fire-and-forget (no await, catch silently)
- Offline fallback: if fetch fails, return empty results (game uses `word.association` from words.json)

Add a `filterOffensiveContent(text: string): boolean` function in the same file. Contains a Set of ~200 Hebrew + English banned words/phrases. Returns true if offensive. Used by WordHackPhase2 before calling `submitAssociation`.

## Chunk 3: Game Components (3 Phases)

### Task 4: WordHackMode — Main Component + State Machine

**Files:**
- Create: `src/features/vocabulary/games/WordHackMode.tsx`

- [ ] **Step 1: Create WordHackMode.tsx**

This is the main game component that manages the 3-phase state machine.

**State machine:**
```
loading → phase1 → phase2 → phase3 → results
```

**On mount:**
1. Use `useWordSelector` to pick 10 words (mode: 'mixed')
2. Call `fetchAssociationsBatch(wordIds)` to preload all associations
3. Transition to `phase1`

**Phase transitions:**
- Phase 1 complete (all 10 words viewed) → phase2
- Phase 2 complete (all 10 words rated) → phase3
- Phase 3 complete (all 10 words quizzed) → results (via `GameResultScreen`)

**Render:** Wraps in `GameShell` portal. Shows progress bar (3 segments). Renders `WordHackPhase1`, `WordHackPhase2`, or `WordHackPhase3` based on current phase.

**Hooks used:**
- `useGameSession({ gameId: 'wordHack', totalQuestions: selectedWords.length, words })` — for XP tracking
- `useWordSelector(words, { count: missionWordCount || 10, mode: 'mixed' })` — for word selection

**Props passed to phases:**
- `words`: the selected 10 words
- `associations`: Record<wordId, WordAssociations> from API
- `onWordComplete(wordIndex)`: marks word done in current phase
- `onPhaseComplete()`: triggers transition to next phase

**Extra state for GameResultScreen:**
- `ratingsGiven: number` — count of ratings submitted in Phase 2
- `associationsCreated: number` — count of student associations submitted

### Task 7: Phase 3 — Quiz (בוחן)

**Files:**
- Create: `src/features/vocabulary/games/components/WordHackPhase3.tsx`

- [ ] **Step 1: Create WordHackPhase3.tsx**

Multiple-choice quiz on the 10 words. Pattern similar to existing `RecallRush`.

**For each word:**
- English word only (large, 26px, bold, white). No translation, no image.
- "???" placeholder for translation
- Two hint buttons:
  - "💡 אסוציאציה" — reveals the association, costs 5 XP
  - "🖼️ תמונה" — reveals the image, costs 3 XP. Hidden if `!hasWordImage(word.id)`
- 4 answer options (Hebrew translations): 1 correct + 3 distractors

**Distractor selection:**
- From same unit (`word.unit`) when possible
- Fisher-Yates shuffle the 4 options
- Ensure no duplicate translations

**Scoring:**
- Correct, no hints: +15 XP
- Correct, image hint only: +12 XP
- Correct, association hint only: +10 XP
- Correct, both hints: +7 XP
- Wrong: 0 XP
- Track via `gameSession.recordCorrect(xpOverride)` or `gameSession.recordWrong(word, selectedAnswer)`

**Answer feedback:**
- Correct: green flash (#4ADE80), check animation, brief show of translation
- Wrong: red flash (#FB7185), shake animation, show correct answer for 1.5s
- Wrong words added to `wrongWords` list (shown in GameResultScreen)

**Timer:** show elapsed time per word (cosmetic, not penalty)

**When all 10 done:** call `onComplete(result)` with the `GameResult` from `useGameSession`

**Verification:** Full game playable end-to-end. `npm run build` passes.

### Task 9: Import Script for D1

**Files:**
- Create: `scripts/import-associations.mjs`

- [ ] **Step 1: Create import script**

Takes two inputs:
1. `src/data/vocabulary/words.json` — original associations (727 words with `word.association`)
2. `scripts/generated-associations.json` — AI-generated associations (from generate script)

Sends to `POST /admin/import` endpoint on the Worker.

```
node scripts/import-associations.mjs --admin-token=<TOKEN> --worker-url=https://znk-associations.<account>.workers.dev
```

Process:
1. Read words.json, filter words with `association` field → source='original'
2. Read generated-associations.json → source='ai'
3. Combine into single array
4. Batch POST to `/admin/import` (100 per batch)
5. Log progress + failures

### Task 10: Offline Fallback + Edge Cases

**Files:**
- Modify: `src/features/vocabulary/games/WordHackMode.tsx`

- [ ] **Step 1: Handle Worker-unavailable fallback**

If `fetchAssociationsBatch` fails (network error, Worker down):
- Phase 1: Show `word.association` from words.json (the local field)
- Phase 2: Skip rating/community features, show local association only
- Phase 3: Quiz works normally (doesn't need API)
- Queue failed ratings/associations in localStorage for retry on next session

- [ ] **Step 2: Handle missing associations**

Some words may have no association at all (neither original, AI, nor community):
- Phase 1: Show word without association card (just word + image + example)
- Phase 2: Skip rating, but still allow creating a custom one
- Phase 3: Quiz works normally

- [ ] **Step 3: Handle missing images**

Use `hasWordImage(word.id)` checks:
- Phase 1: If no image, skip the image section (association card fills the space)
- Phase 2: If no image, hide thumbnail
- Phase 3: Hide "🖼️ תמונה" hint button if no image
